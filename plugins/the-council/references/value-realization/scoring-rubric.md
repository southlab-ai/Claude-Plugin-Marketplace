# Value Realization Scoring Rubric

Four-dimension assessment framework. Each dimension is rated green/yellow/red based on observable criteria.

---

## 1. Value Clarity

How clearly can users articulate the specific outcome they get from the product?

| Rating | Criteria |
|--------|----------|
| **Green** | Users articulate specific outcomes in their own words. They say "I use X to achieve Y" with concrete Y. Value proposition is self-evident from first interaction. |
| **Yellow** | Users understand the general category of value but cannot name specific outcomes. They say "it's a tool for Z" without explaining what Z accomplishes for them personally. |
| **Red** | Users describe features, not outcomes. They say "it has A, B, C features" but cannot connect those features to personal results. Value proposition requires external explanation. |

**Indicators**: User interview language, app store review content, word-of-mouth referral phrasing, onboarding completion rates.

---

## 2. Value Timeline

How well does the timing of value delivery match the product's nature and user expectations?

| Rating | Criteria |
|--------|----------|
| **Green** | Timing matches product nature and user expectations. Immediate-value products deliver immediately. Investment products set clear expectations upfront and deliver on schedule. Users feel the timeline is natural. |
| **Yellow** | Gap between expected and actual value timeline. Users expect faster results than the product delivers, or the product delivers value that users don't yet recognize. Requires patience or reframing. |
| **Red** | Timing conflicts with user expectations. Product requires investment but promises immediacy, or delivers immediately but users expected depth. Mismatch causes abandonment before value materializes. |

**Indicators**: Time-to-first-value metrics, day-1/7/30 retention curves, churn timing patterns, support ticket themes in first week.

---

## 3. Value Perception

How visible and tangible is the value to users once delivered?

| Rating | Criteria |
|--------|----------|
| **Green** | Users can point to specific evidence of value received. Metrics, artifacts, visible changes, or clear before/after comparisons exist. Value is self-documenting. |
| **Yellow** | Value is partially visible but requires interpretation. Users sense improvement but need dashboards, reports, or reflection to confirm it. Some value is felt but not provable. |
| **Red** | Value is invisible to users even when delivered. The product works but users cannot see or measure the difference. No tangible proof exists. Users question whether the product does anything. |

**Indicators**: Feature usage that generates visible artifacts, user-initiated sharing/screenshots, NPS qualitative responses, renewal/upgrade decision reasoning.

---

## 4. Value Discovery

How do users find and recognize value beyond the initial use case?

| Rating | Criteria |
|--------|----------|
| **Green** | Users experience natural recognition or an "aha moment" within minutes of engagement. Discovery happens through normal use, not instruction. Users find additional value beyond what they expected. |
| **Yellow** | Users need guidance, tutorials, or onboarding flows to discover value. Left alone, they use only surface features. With help, they find deeper value. Discovery is possible but not organic. |
| **Red** | No clear discovery path exists. Users complete onboarding but never find the core value. They abandon before recognizing differentiation from alternatives. The product fails to distinguish itself. |

**Indicators**: Feature adoption breadth over time, organic use-case expansion, tutorial completion vs. feature discovery without tutorials, time-to-aha measurement.

---

## Usage Notes

- These criteria are **reference anchors**, not rigid rules. Real products exist on a spectrum.
- **Context determines judgment.** A "yellow" timeline for a fitness app (results take weeks) might be perfectly acceptable if expectations are set correctly.
- **Your analysis takes precedence over indicators.** Indicators suggest where to look; they do not determine the rating. A product with poor retention metrics might still have green clarity if the users who stay can articulate value perfectly.
- When assessing a product, evaluate each dimension independently. A product can be green on clarity and red on discovery simultaneously.
